jsoncpp open source 라이브러리를 프로젝트에 인클루드 하는법

visual studio IDE 기준,
프로젝트 설정 - 
C/C++ 탭 -
일반 탭 - 
추가 포함 디렉터리 설정을 다음과 같이 바꿔준다:
	$(ProjectDir)\include\jsoncpp-1.8.4\include 

그 후 라이브러리를 링크하기 위해
프로젝트 설정 -

일반 탭 - 
추가 라이브러리 디렉터리 설정을 다음과 같이 바꿔준다:
	$(ProjectDir)\include\jsoncpp-1.8.4\makefiles\msvc2010\Debug

빌드 오류 [1] --
1>VS ERROR LNK2019 : _main 외부 기호(참조 위치: "int __cdecl invoke_main(void)" (?invoke_main@@YAHXZ) 함수)에서 확인하지 못했습니다. --

1. 프로그램 진입점을 제대로 작성하지 않아서인 경우가 있다.
   main 함수를 제대로 작성했는지 확인

2. visual studio 설정 변경으로 인한 오류인 경우도 있다.
   하위 시스템을 콘솔로 고정토록 하는 것인데, 콘솔의 진입점은 main이고 WINDOWS 데스크톱 프로그래밍의 진입점은 WinMain이기 때문이다.
   전 프로젝트가 Windows API 프로그래밍 이었기 때문에 설정 변경이 있을 수 있으니 설정 변경 후 빌드 진행한다.

   - [프로젝트 설정] - [구성 속성] - [링커] - [시스템] 으로 진입한다.
   - 하위 시스템 설정이 "창(/SUBSYSTEM:WINDOWS)" 으로 되어 있다면
   - "콘솔(/SUBSYSTEM:CONSOLE)" 또는 "설정 안함" 으로 변경한다.

빌드 오류 [2] --
1>test_json.cpp
1>lib_json.lib(json_value.obj) : error LNK2038: 'RuntimeLibrary'에 대해 불일치가 검색되었습니다. 'MTd_StaticDebug' 값이 'MDd_DynamicDebug'(test_json.obj에 위치) 값과 일치하지 않습니다.
1>lib_json.lib(json_writer.obj) : error LNK2038: 'RuntimeLibrary'에 대해 불일치가 검색되었습니다. 'MTd_StaticDebug' 값이 'MDd_DynamicDebug'(test_json.obj에 위치) 값과 일치하지 않습니다.

및 LNK2005, LNK2038 등의 오류가 발생한다면 적용

정적으로 프로젝트를 라이브러리와 링크하려고 하는데 (.lib 확장자 파일을 링크)
visual studio 프로젝트의 링커 설정은 동적으로 링크하려고 하여 설정의 불일치가 발생한 경우이다. (링커가 런타임에 .dll 확장자 파일과 링크)
(예시 프로그램 소스에 #pragma comment (lib, "lib_json.lib") 와 같이 "lib_json.lib"를 링크할 것을 명령했기 때문이다.)

visual studio에게 정적으로 링크하라고 명령하기 위해 다음과 같이 설정해준다:
   - [프로젝트 설정] - [구성 속성] - [C/C++] - [코드 생성]으로 진입한다.
   - 런타임 라이브러리 설정이 "다중 스레드 디버그 DLL(/MDd)"로 되어 있다면
   - "다중 스레드 디버그(/MTd)" 로 변경한다.
